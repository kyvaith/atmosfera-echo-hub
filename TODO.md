# Atmosfera Echo Hub TODO

- Investigate a lower-power XMOS idle mode for Satellite1-style voice handling: keep wake word quality, but reduce or suspend the heavier AEC/NS/AGC pipeline until an active voice session or playback reference requires it.
- Revisit the manual direct framebuffer snapshot compositor for tile swipes after the broader LVGL performance work. The target baseline is the version after the excessive touch-handler work was removed, but before the manual direct compositor was disabled: swipe UX was good there, and that exact combination is worth restoring once LVGL/render throughput is improved.
- Evaluate replacing the custom home-screen swipe pager with ESPHome/LVGL `tileview`: keep one-page-per-swipe behavior, preserve the up-swipe app-close gesture, and compare invalidation/FPS against the current manual `controls_root_*` positioning before adopting it.
- Experiment with a generic LVGL virtualized/recycled list mechanism: keep only the visible row widgets alive and update their content/position from a backing data model while scrolling. This must be designed as a reusable collection primitive for settings, menus, pickers, and future long lists, not as a one-off settings-page workaround.
- Harden the full LVGL compile-time profiler path (`profiler: true`) so it is reliable on-device: prevent boot loops/panics, keep a reproducible settings-scroll capture flow, and document when to use compile-time profiling versus the lightweight runtime perf logger.
- Calibrate display backlight limits: brightness below roughly 70% currently turns the screen fully off. Measure the real usable PWM range for this panel/backlight circuit, then set safe min/max thresholds and remap the UI brightness slider so low values remain visible.
- Evaluate an ESP32-P4 bridge for LVGL's DMA2D draw backend. The upstream LVGL DMA2D backend is currently STM32-register based, so enabling it on this hardware needs a real ESP-IDF/PPA/DMA2D HAL adapter rather than only unfiltering `src/draw/dma2d`.
- Revisit dual-core display flushing as a driver-level experiment: keep LVGL rendering into small internal-RAM partial buffers while a separate task/core copies into the inactive full MIPI framebuffer, swaps at frame end, and syncs the idle buffer. The first LVGL-component prototype bootlooped before logger initialization, so the next attempt should be gated behind the MIPI DSI driver and introduced in smaller link/runtime steps.
