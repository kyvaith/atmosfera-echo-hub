# Atmosfera Echo Hub TODO

- Revisit and harden the page snapshot compositor for tile swipes after the broader LVGL performance work. The target baseline is the version after the excessive touch-handler work was removed, but before the manual direct compositor was disabled: swipe UX was good there, and that exact combination is worth restoring once LVGL/render throughput is improved.
- Look for a high-quality LVGL swipe/pager reference implementation and compare it against the current home-screen swipe behavior. The current gesture still does not feel natural enough, especially around thresholding, inertia, and the final settle animation.
- Test keeping LVGL swipe/page snapshots in PSRAM as compressed JPEG payloads, using ESP32-P4 hardware JPEG encode/decode where possible. Compare memory pressure, decode latency, cache-sync cost, and visual artifacts against raw RGB888 panorama snapshots before deciding whether compressed snapshot storage is worth the complexity.
- Add a user/developer-facing switch for the snapshot compositor JPEG compression path, so compressed snapshots can be enabled or disabled without rebuilding experimental code paths by hand.
- Split JPEG snapshot compression and the snapshot compositor into separate components/backends. The compositor should own capture, cache lifecycle, transitions, and restoration of live LVGL, while JPEG compression/decompression should be a pluggable storage/codec layer that can later be tested, disabled, or replaced independently.
- Move the ESP-IDF JPEG/DMA2D patching currently living in `esphome/components/mipi_dsi/patch_espidf_dsi_dma2d.py` into the future JPEG hardware encoder/decoder work. That patch belongs with the JPEG codec PR, not with the MIPI DSI display-driver PR, because it changes JPEG RGB CSC selection and DMA2D YUV-to-RGB conversion behavior.
- Evaluate replacing the custom home-screen swipe pager with ESPHome/LVGL `tileview`: keep one-page-per-swipe behavior, preserve the up-swipe app-close gesture, and compare invalidation/FPS against the current manual `controls_root_*` positioning before adopting it.
- Experiment with a generic LVGL virtualized/recycled list mechanism: keep only the visible row widgets alive and update their content/position from a backing data model while scrolling. This must be designed as a reusable collection primitive for settings, menus, pickers, and future long lists, not as a one-off settings-page workaround.

- Fix minimum and maximum display brightness limits: brightness below roughly 70% currently turns the screen fully off. Measure the real usable PWM range for this panel/backlight circuit, then set safe min/max thresholds and remap the UI brightness slider so low values remain visible while full brightness stays within the panel's comfortable range.
- Investigate screen flicker during OTA updates. Consider turning the display/backlight off for the OTA window, or showing a static low-bandwidth OTA screen, so flash writes/network activity cannot produce visible scanout artifacts.
- Improve display wake-up sequencing after sleep: delay backlight enable until LVGL has had time to redraw the active page, and make the wake fade-in roughly twice as long so the backlight ramp does not expose stuttered rendering.
- Investigate and fix the one-time post-boot scheduler warning: `<null> took a long time for an operation (...)`. It likely comes from a generated YAML `DelayAction` or anonymous scheduled callback in the boot sequence; identify the exact delayed action, give it attributable context if possible, and reduce the blocking work after boot.
- Investigate the remaining occasional blue screen flash that appears every few minutes. Check for rare DSI underruns, bandwidth spikes, and timing-margin issues; test whether lowering display bandwidth/lane bitrate or pixel clock improves stability without hurting responsiveness.

- Add a configurable page snapshot refresh cadence for snapshot-backed pages. Generate the next snapshot in the background and only replace the old snapshot after the new one is fully rendered, so a swipe that starts mid-refresh can still animate smoothly without a blank or stalled frame.
- Redesign the current full-screen snapshot compositor into a generic upstreamable LVGL snapshot transition/cache layer. The target design should support snapshotting any `lv_obj_t` subtree, not only full-screen pages; derive bounds from object coordinates instead of hardcoded 800x800 geometry; use the display color format/byte order instead of hardcoded RGB888; expose cache limits, explicit invalidation, and transition actions from YAML; keep the hardware-specific ESP32-P4 PPA/DMA2D/direct-framebuffer path as an optional backend rather than the core abstraction. The core PR-worthy concept is: freeze an expensive LVGL subtree into a bitmap, animate/scroll that bitmap cheaply, then restore the live LVGL tree cleanly at the end.
